## Summary

**Verdict: Minor Issues.** The test file verifies basic manifold operations (projection, tangent projection, exp/log roundtrip) for `Sphere`, `Torus`, `Paraboloid`, and `EuclideanSpace`. All asserted mathematical properties are correct — no test checks for a wrong expected value. However, coverage is thin: there are no tests for curvature, fundamental forms, parallel transport, or the Hyperboloid manifold. Several assertions that would strengthen mathematical verification are absent entirely. Tolerances are generally appropriate for float64 arithmetic.

## Formula-by-Formula Audit

### 1. `test_sphere_operations` — Projection norm check (line 23)

- **Spec Reference:** `spec/MATH_SPEC.md` § 4 (Surface Formulations); `Sphere.project` normalises to $\|x\| = r$.
- **Implementation:**
  ```python
  norms = jnp.linalg.norm(pts, axis=-1)
  np.testing.assert_allclose(norms, 2.0, atol=1e-5)
  ```
- **Verdict:** CORRECT
- **Notes:** For $S^n(r=2)$, all points must satisfy $\|x\| = 2$. Tolerance `1e-5` is well above machine epsilon for float64 ($\approx 10^{-16}$) and appropriate for this test.

### 2. `test_sphere_operations` — Tangent orthogonality (line 33)

- **Spec Reference:** $T_x S^n(r) = \{v : \langle v, x \rangle = 0\}$
- **Implementation:**
  ```python

  self.assertAlmostEqual(jnp.dot(v_tan, x_proj), 0.0, places=5)
  ```
- **Verdict:** CORRECT
- **Notes:** Verifies the defining property of the tangent space. `places=5` corresponds to tolerance $5 \times 10^{-6}$, appropriate.

### 3. `test_sphere_operations` — Retraction stays on manifold (line 36)

- **Spec Reference:** $\exp_x(v) \in S^n(r) \implies \|\exp_x(v)\| = r$
- **Implementation:**
  ```python
  y = sphere.retract(x_proj, v_tan)
  self.assertAlmostEqual(jnp.linalg.norm(y), 2.0, places=5)
  ```
- **Verdict:** CORRECT
- **Notes:** Since `retract` delegates to `exp_map`, this verifies the analytically proven identity $\|\cos\theta \cdot x + \frac{\sin\theta}{\theta} v\| = r$ (where $\langle x, v \rangle = 0$).

---

### 4. `test_sphere_operations` — Log/Exp roundtrip (lines 36–39)

- **Spec Reference:** $\log_x(\exp_x(v)) = v$ for all $v \in T_x S^n(r)$
- **Implementation:**
  ```python
  y = sphere.retract(x_proj, v_tan)
  v_log = sphere.log_map(x_proj, y)
  np.testing.assert_allclose(v_log, v_tan, atol=1e-5)
  ```
- **Verdict:** CORRECT
- **Notes:** This is the critical inverse relationship. The tolerance `1e-5` is adequate given the clipping in `log_map` ($\cos\theta$ clipped by `GRAD_EPS = 1e-12`). For this particular test, $v_\text{tan}$ is small (derived from `[1.0, -1.0, 0.0]` projected to $T_x S^2(2)$), so $\theta$ is moderate and roundtrip error is well below $10^{-5}$.

### 7. `test_torus_operations` — Sample/project consistency (lines 62–63)

### 9. `test_euclidean_space` — Flat geometry identities (lines 75–84)

- **Spec Reference:** `spec/MATH_SPEC.md` § 4 hierarchy table: Euclidean space has $G^i = 0$, $\Gamma^i_{jk} = 0$; $\exp_x(v) = x + v$; $\log_x(y) = y - x$; $\text{project} = \text{id}$.
- **Implementation:**

  ```python
  np.testing.assert_allclose(euc.exp_map(x, v), x + v)

  ```
- **Verdict:** CORRECT
- **Notes:** These are trivially correct for $\mathbb{R}^n$. Default `atol=0` (exact match up to floating point), appropriate since the operations are exact additions.

## Coverage Gaps (Mathematical Properties Not Tested)

### G1. Hyperboloid manifold — no tests

- **Severity:** WARNING
- **File:** `tests/test_surfaces.py` (entire file)
- **Notes:** The `Hyperboloid` class implements exp/log maps, parallel transport, projection, and a metric tensor — all with non-trivial Minkowski-signature formulas. None of these are tested.
- **Recommended Action:** Add tests verifying:
  - $\langle x, x \rangle_L = -1$ for projected/sampled points.
  - $\langle x, v \rangle_L = 0$ for tangent vectors.
  - $\langle \exp_x(v), \exp_x(v) \rangle_L = -1$.
  - $\log_x(\exp_x(v)) = v$ roundtrip.
  - $\langle y, P_{x \to y}(v) \rangle_L = 0$ and $\langle P(v), P(v) \rangle_L = \langle v, v \rangle_L$ for parallel transport.

---

### G2. Parallel transport — no tests for any manifold

- **Severity:** WARNING
- **File:** `tests/test_surfaces.py` (entire file)
- **Notes:** `Sphere.parallel_transport` and `Hyperboloid.parallel_transport` implement closed-form Levi-Civita transport (reviewed and verified analytically in `reviews/math/surfaces.md` §6, §19). The key mathematical properties — tangency ($\langle y, P(v) \rangle = 0$), norm preservation ($\|P(v)\| = \|v\|$), and agreement with the Berwald connection (spec § 3.2) — are never tested.
- **Recommended Action:** For the sphere, add:
  ```python
  pt = sphere.parallel_transport(x, y, v_tan)
  self.assertAlmostEqual(jnp.dot(pt, y), 0.0, ...)  # tangency
  self.assertAlmostEqual(jnp.linalg.norm(pt), jnp.linalg.norm(v_tan), ...)  # isometry
  ```

---

### G3. Curvature — no tests

- **Severity:** NOTE
- **File:** `tests/test_surfaces.py` (entire file)
- **Notes:** The file is named `test_surfaces.py` but does not test any curvature-related quantities. For a sphere $S^n(r)$, the Gaussian curvature is $K = 1/r^2$ (constant). For a torus $T^2(R, r)$ at angle $v$, $K = \cos v / (r(R + r\cos v))$. These are standard textbook values that could be checked against the library's curvature module (if `src/ham/geometry/curvature.py` exists and applies to surfaces).
- **Recommended Action:** If curvature computation is available on these surface types, add tests with known analytical values.

### G4. Sphere exp/log — only small-angle regime tested

- **Severity:** WARNING
- **File:** `tests/test_surfaces.py:36–39`
- **Notes:** The `v_tan` used in the roundtrip test is derived from `[1.0, -1.0, 0.0]` projected onto $T_{x_{\text{proj}}} S^2(2)$. After projection, $\|v_\text{tan}\| / r$ is moderate ($\theta \approx 0.4$ rad). The test does not exercise:
  - **Near-identity:** $\theta \approx 0$ (Taylor branch $\theta < \texttt{TAYLOR\_EPS} = 10^{-6}$).
  - **Large angle:** $\theta \approx \pi$ (near cut locus / antipodal point), where `log_map` is most fragile due to the $\arccos$ clipping.
  - **Exact quarter-turn:** $\theta = \pi/2$, a canonical test point.
- **Recommended Action:** Add test cases with $v_\text{tan}$ scaled so that $\theta \in \{10^{-8}, \pi/4, \pi/2, \pi - 0.01\}$ and verify the roundtrip.

### G5. Torus — no exp/log or tangent tests

- **Severity:** WARNING
- **File:** `tests/test_surfaces.py:60–63`
- **Notes:** The torus test only checks that `project(random_sample(...))` is the identity. It does not test:
  - `to_tangent` produces vectors orthogonal to the surface normal.
  - `exp_map` (projected retraction) keeps points on the torus.
  - `log_map` (base-class fallback) roundtrips with `exp_map` for small displacements.
- **Recommended Action:** Add at minimum:
  ```python
  x = pts[0]
  v = jax.random.normal(key, (3,))
  v_tan = torus.to_tangent(x, v)
  y = torus.exp_map(x, v_tan * 0.01)
  proj_y = torus.project(y)
  np.testing.assert_allclose(y, proj_y, atol=1e-5)
  ```

## Open Questions

1. **Missing Hyperboloid tests:** The `Hyperboloid` class is the most mathematically complex surface in the library (Minkowski signature, hyperbolic trig, custom JVP rules). Its absence from `test_surfaces.py` is a significant coverage gap. Is it tested elsewhere?

2. **No curvature tests:** If the library computes Gaussian or sectional curvature on these surfaces (via `src/ham/geometry/curvature.py`), those known-value tests belong here. Is curvature tested in `tests/test_metric.py` or similar?

3. **No first/second fundamental form tests:** The user request mentions fundamental forms, but neither the test file nor the `surfaces.py` source explicitly compute $I = g_{ij}\,du^i\,du^j$ or $II = b_{ij}\,du^i\,du^j$. The fundamental tensor $g_{ij}$ is computed via autodiff from the energy in the Finsler layer, not at the surface level. Are fundamental forms tested in a different module?

4. **Deterministic random seed:** All tests use `PRNGKey(42)`. This is fine for reproducibility, but a single fixed seed may not exercise edge-case configurations (e.g., points near the poles of $S^n$). Consider parameterised tests with multiple seeds if edge-case coverage is desired.
