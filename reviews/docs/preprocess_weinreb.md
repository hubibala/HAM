# Documentation Review: `examples/preprocess_weinreb.py`

**Reviewer:** Doc Reviewer Agent
**Date:** 2026-05-15

## Summary

Overall documentation quality: **adequate**.

The script has a meaningful module-level docstring that explains the pseudo-velocity derivation strategy and the anti-circularity argument. The six numbered sections are clearly delineated with header comments. However, the docstring title (`preprocess_weinreb_spring.py`) does not match the actual filename (`preprocess_weinreb.py`), several output files are not mentioned in the docstring, the `inspect()` helper is undocumented, the input data schema is only partially described, and there is no reference to the original Weinreb publication or to `spec/ARCH_SPEC.md § 5`.

## Spec Alignment Notes

- `spec/ARCH_SPEC.md` line 198 describes the Weinreb application ("GeometricVAE + DataDrivenPullbackRanders trained on Weinreb hematopoiesis data. Experiments H1-H4 …"). This preprocessing script is the data entry point for that entire pipeline but is not mentioned in the spec. The script itself makes no back-reference to the spec.
- The module docstring (line 11) describes a Gaussian-kernel-weighted velocity computation, but the implementation (lines 160–168) computes an unweighted mean. This is an **INACCURATE** discrepancy between documentation and code (Finding #11).
- The output file `data/weinreb_lineage_triples.npy` referenced by downstream scripts (`weinreb_experiment.py:636`, `weinreb_vae.py:894`) is **never produced** by this preprocessing script. The script generates `weinreb_train_triples.npy` and `weinreb_test_triples.npy` instead. This mismatch is also flagged in `reviews/science/weinreb_experiment.md:125` but should be noted here as a documentation gap: neither the preprocessing script nor the downstream scripts document which triple file is the canonical one.
- `spec/MATH_SPEC.md` contains no mention of the Weinreb preprocessing. The pseudo-velocity derivation (clonal mean displacement in PCA space) is a methodological choice that arguably belongs in the math spec or at minimum should be cross-referenced from the script docstring.
