# Documentation Review: `examples/demo_weinreb_vis.py`

**Reviewer:** Doc Reviewer Agent  
**Date:** 2026-05-15

## Summary

Overall documentation quality: **needs work**.

The script builds a synthetic differentiation tree on the Hyperboloid and visualises it on the Poincaré disk — a compelling tutorial for the HAM biological pipeline. However, it has **no module-level docstring**, several key algorithmic steps lack inline comments, and the mathematical context (Minkowski model, exponential/logarithmic maps) is never connected to the spec. A mathematician reading the script cannot tell which formulas are being invoked; an ML engineer cannot tell what the output represents biologically without prior knowledge of the Weinreb dataset. The `generate_synthetic_tree` function has a minimal docstring that omits `Args`, `Returns`, and mathematical context. The `main` function has no docstring at all.

## Spec Alignment Notes

1. **Filename vs. content mismatch.** The file is named `demo_weinreb_vis.py` and the output is `weinreb_hyperbolic_vis.png`, strongly implying a connection to the Weinreb hematopoiesis dataset (`spec/ARCH_SPEC.md § 5`). In reality the script generates a purely synthetic branching tree with no real biological data. The script never explains this relationship — it should state that it is a *synthetic analogue* of the Weinreb pipeline, demonstrating the visualisation tooling without requiring the preprocessed dataset.

2. **Hyperboloid formulas.** The script uses `Hyperboloid.exp_map` and `Hyperboloid.log_map` which implement the exact $\cosh/\sinh$ maps described in `spec/MATH_SPEC.md § 4.1` ("The Hyperboloid models the upper sheet in Minkowski space and features exact cosh/sinh exponential and logarithmic maps"). Neither the docstrings nor the inline comments reference the spec or state the formulas, leaving the mathematical audience without a connection to the theory.

3. **Tangent projection.** `Hyperboloid.to_tangent` implements the Minkowski-orthogonal projection $v \mapsto v + \langle x, v \rangle_L \, x$ (where $\langle \cdot, \cdot \rangle_L$ is the Lorentzian inner product). The script calls it on line 34 but only has the comment "tangent_dir = manifold.to_tangent(...)". The mathematical operation deserves a brief note.

4. **Poincaré projection.** The sterographic projection $y = x_{\text{spatial}} / (1 + x_0)$ is implemented in `src/ham/vis/hyperbolic.py` and documented in the `project_to_poincare` docstring, but `spec/MATH_SPEC.md` does not include a dedicated section on this projection. The demo should at minimum reference the vis module where the formula lives.

5. **Wind-field convention.** The "wind" constructed on line 66 via Log maps follows the spec convention where wind vectors represent directional transport on the manifold (`spec/MATH_SPEC.md § 5`). However, the script never calls this a Randers wind or connects it to the Zermelo navigation framework — it is simply parent-to-child tangent vectors plotted as arrows.
