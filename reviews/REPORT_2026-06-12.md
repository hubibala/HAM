## 4. Fixes implemented in this pass (verified)

### 4.1 — CRITICAL: VAE KL was missing the location (mean) term
**File:** `research/weinreb/bio/vae.py` — `WrappedNormal.kl_divergence_std_normal`
**Problem:** the divergence was `Σ(½σ² − log σ − ½)`, i.e. `KL(N(0,σ²)‖N(0,1))` with the mean assumed
at the origin. The posterior **mean was completely unregularized**, removing the prior's control over
latent *location* and inviting unbounded drift / posterior issues.
**Fix:** added the manifold-correct location term `½·d_M(origin, μ)² = ½‖log_o μ‖²` (Minkowski norm on
the hyperboloid; reduces to `½‖μ‖²` on flat space). Factored the per-manifold origin into a shared
`_prior_origin()` helper so `sample()` and the KL agree. Reference: Nagano et al., ICML 2019.
**Verification:** new regression tests in `research/weinreb/tests/test_hyperbolic_vae.py`
(`test_kl_regularizes_mean`, `test_kl_reduces_to_euclidean`) — KL is ~0 at the pole with unit scale,
grows monotonically with geodesic distance, and equals `½‖μ‖²` exactly on `EuclideanSpace`. All 7 VAE
tests pass.

### 4.2 — CRITICAL: RNA velocity never projected into PCA latent → wind signal silently zero
**File:** `research/weinreb/bio/data.py` — `DataLoader.preprocess`
**Problem:** `scv.tl.velocity` writes a *gene-space* `layers['velocity']`; nothing populated
`obsm['velocity_pca']`, so `get_jax_data` fell through to `V = zeros_like(X)`. With `V≡0` the
data-driven Randers wind is zero everywhere and the **entire Finsler/directional advantage collapses to
a symmetric Riemannian metric** — exactly the framework's headline capability, silently disabled.
**Fix:** after PCA + velocity, linearly project the gene-space velocity through the PCA loadings
`obsm['velocity_pca'] = nan_to_num(velocity) @ varm['PCs']` (the PCA basis is linear, so velocities map
by the same loadings; genes without a fit carry NaN and are treated as zero drift). Guarded on the
presence of `layers['velocity']` and `varm['PCs']`.
**Note:** cannot be end-to-end verified here (requires the Weinreb `.h5ad` + scvelo); the projection is
the standard, mathematically correct linear map and is byte-compile-checked.

### 4.3 — Portability: Windows console crash in the test suite
**File:** `tests/test_geodesic_learning.py:380` — a `print()` containing a literal `→` raised
`UnicodeEncodeError` on the Windows cp1252 console, failing the test (and, by the same mechanism, any
experiment/example run on a non-UTF-8 terminal). Replaced with ASCII `->`. `test_geodesic_learning`
now passes on Windows (5/5).

### 4.4 — Dead import
`research/wildfire/experiment_wildfire_geometric.py:50` imported the deprecated `scalar_curvature`
without using it. Removed.

## 5. Residual findings (not yet implemented — require judgement or data)

### Core framework — WARNING level
- **W-EL** (`training/losses.py:287` `EulerLagrangeResidualLoss`): the residual differentiates a
  *hand-rolled* smoothed Randers Lagrangian `√(vᵀHv+ε²) − ⟨W,v⟩_H` rebuilt from `zermelo_data`, which is
  **not** the metric the model actually optimizes (the true `metric_fn` uses the Zermelo discriminant
  form with the `1/λ` factor). The rigorous form is to differentiate `model.metric.energy` directly via
  autodiff (the spray machinery in `metric.py` already does exactly this). Recommend replacing the
  hand-coded `L_smooth` with `lambda z,v: model.metric.energy(z, v)` and reusing `metric.spray`'s
  EL-residual assembly. Low risk, improves correctness and removes ~30 lines.
- **W-GS** (`losses.py:84` `GeodesicSprayLoss`): penalizing `‖spray‖²→0` biases the metric toward
  *flatness* rather than toward "observed paths are geodesics." Prefer an E-L residual on observed
  trajectories (above) as the geodesic-consistency signal.
- **W-MK** (`losses.py:77`, `vae.py:134`): `ZermeloAlignmentLoss`, `ContrastiveAlignmentLoss`, and
  `GeometricVAE.loss_fn` call hyperboloid-private `manifold._minkowski_norm/_dot`. These silently
  require a Hyperboloid manifold; they will `AttributeError` on Euclidean/Sphere. Either gate on
  `isinstance(manifold, Hyperboloid)` or add a generic `tangent_norm`/`tangent_dot` to the `Manifold`
  ABC. (The `vae.loss_fn` is legacy "backwards-compat"; the modular-loss path is preferred.)
- **W-RAND** (`zoo/randers.py:53`): the wind squash is `scale = where(w_norm<0.5, 1, max_speed·tanh(w_norm)/w_norm)`.
  At `w_norm=0.5` the squashed magnitude jumps from `0.5` to `≈0.462`, a small **C⁰-but-not-C¹**
  discontinuity in the wind field (hence in `F` and its gradients). Replace the hard branch with a single
  smooth squash (e.g. always `max_speed·tanh(w_norm)/w_norm`, or a smooth blend) to keep `F∈C¹`.

### Wildfire experiments — science gaps
- **WF-JAC** (`experiment_wildfire_geometric.py:103`): "Jacobi divergence" = mean inter-ray distance,
  which **rises monotonically even in flat Euclidean space** (a geodesic fan always opens radially). As
  plotted it cannot distinguish "unstable spread" from trivial radial opening. Normalize against the
  flat-metric baseline (ratio to the Euclidean fan at the same radius) or compute the actual Jacobi
  field / geodesic-deviation operator from the curvature tensor (`curvature.py` already provides it).
- **WF-REAL** (`experiment_wildfire_geometric.py:541`): the real-data path is a stub
  (`implement the load_checkpoint() call`). Only `--synthetic` works end-to-end. For a publication
  figure the W1(flat)/W2(mesh) training → checkpoint → W3(analysis) pipeline must connect, with a
  held-out fire set.
- The flat/mesh training experiments and their IoU/Pearson evaluation were not re-audited this pass;
  the May report's RB1–RB5 (arrival-time scale / IoU normalization / IFT conditioning) appear addressed
  in `losses.py` (curriculum Pearson→relMSE, documented `stop_gradient` only on the normalizer), but
  this should be re-verified against the current `experiment_wildfire_flat.py`.

### Weinreb experiments — methodology
- **WB-LEAK** (`experiment_h2_directional.py:38`): `attach_datadriven_randers_metric` builds the wind
  from anchors sampled across **all** cells (the comment says "training clones only" but `valid_mask`
  is merely `‖v‖>1e-6`). The same `X_norm` indexes the `test_triples`, so wind at a test point can be
  interpolated from anchors that *are* the test cells → **transductive leakage**. Enforce a clone-level
  train/test split and draw anchors only from the train side. (Same pattern to check in h3/h4.)
- **External baselines:** to substantiate "usefulness of the framework," the directional/predictive
  claims should be benchmarked against established fate-mapping methods (PRESCIENT, Waddington-OT,
  CellRank) on the same split, not only against the internal permuted-wind null.
- h1/h3/h4 and `weinreb_experiment.py` were not re-audited; the May C5/C6/C8/C9 findings (validation-file
  provenance, hardcoded "successfully extracts…" strings, confounded baselines) must be re-checked
  against the current files.

### Tests / portability
- 7 test files (`test_arrival_time_loss`, `test_covariate_randers`, `test_geodesic`,
  `test_implicit_diff_avbd`, `test_terrain`, `test_utils_math`, `test_wildfire_data`) `import pytest`
  while the rest use `unittest`; under the project's `uv` runner (which installs the `dev` extra) these
  pass, but they error under a bare `python -m unittest`. Standardize on one runner (pytest is the
  natural choice) and document `uv run pytest` / `pip install -e ".[dev]"` as the canonical command.
- Non-ASCII characters appear in several test/example `print`s; on non-UTF-8 consoles these crash.
  Either keep output ASCII or set `PYTHONUTF8=1` / reconfigure stdout in a conftest.
- Coverage gaps worth closing (still true): Finsler axiom tests (1-homogeneity `F(x,λv)=λF(x,v)`,
  positive-definiteness of `g_ij`) for learned metrics, and `jit`/`vmap`/`grad`-through-solver tests.
